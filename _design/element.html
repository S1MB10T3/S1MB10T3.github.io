Introduction

Element Finance is a protocol for fixed and variable yield markets. It is a decentralized finance (DeFi) protocol that allows users to deposit assets and earn interest on their deposits. The protocol is built on the Ethereum blockchain and is governed by the Element DAO.

I was brought on to Element as a Design Systems Designer to build out the a new design system for the Element Finance v2 DAPP and the web3 community over all. My position grew to also indiate on the new branding for Element DAO, Element Foundation, and [REDACTED].

Design System

The goal for the design system was to set an industry standard. It can serve as an educational tool and reference for junior-level designers and content contributors in crypto. Explicitly written design systems that span across products, channels and departments aid those who are new to design in our industry and also serve as a reminder for the rest of the contributors.

Inspo

The visual aesthetic for the design system is a call back to early computing/tech. As well as pulling inspo from 50's/60's graphic design.

Guidelines

Rules and suggestions are sometimes more important then the values presented for a design system. Making sure that fellow designers and developers can understand how to use the given text, colors, spacing, and etc< will help keep designs consetant across different pages and applications.

Color

Text about the Solis and how it relates to alechemical elements. Splendor Solis by Salomon Trismosin

Color Blocked

Each of the colors chosen for the theme are at least AA WCAG compliant as long as they used for their inteaded roles. So all Accent, Text, and State colors have a strong contrast will all avaible Foreground and Background options for their repective theme.

All colors have been tested per theme to make sure they look good anywhere they are meant to be applied!

Typography

The Base body text size (16px) is 1rem. So the next step up to LG(18px) is 1.25rem and a step down (14px) would be 0.75rem. This scaling also applies to all text accross the system.

Headings use the Primary Text color and should have a line that's 1px width and 100px length. The line rest at the botton of the heading. To make sure there's always strong visual contrast between the body text and the heading above, especially when using smaller headings such as Heading 6, as shown above, were the text size is the same as the body.

Components

The components had to be easy, varied, and composiable for the other designers on the team. This designers can basically drap and drop then flip some switchs to get the style, color, and size for compoenets in a fly. The system made it so our UI designer can rapidly and realibliy create and change pages faster and with high quaility. Figma's component system is a great tool for this. It allows for crazy amount of properties and inputs as long as it was design right from the start.

Figma's auto layout feature is a must use for any compoenets. This way scaling basically is a non factor and makes it really easy for frontend developers to transalte it to code.
